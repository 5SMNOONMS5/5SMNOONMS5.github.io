前往遊戲開發之路

Nov 15, 2017

一切的開始

一切的開始全是 PM 的一句話 我要把你們帶向混合應用的路線，在經歷三個禮拜陣痛期的洗禮之後，想記錄一下自己這三個禮拜的學習過程。

我們 Team 的背景

目前我們 App Team 總共有四個人。1 位 iOS (就是我拉~~) 和 2 位 Android 外加一位曾經是全端的 PM ，我們全部都沒有遊戲開發的背景。

先定大方向

聽到要開發遊戲的瞬間其實抖了一下，而且只有兩個月的時程包含學習，雖然已經有了後端 Api，而且又在完全沒有任何有經驗的人帶領之下，我們只能從 0 開始。

所以首先要先決定大方向

  1. 到底要用原生 iOS 跟 Android 開發呢，還是用類似 Unity 這種跨平台的 engine 一次 build 兩個平台？

我們的想法是人手不足，採兩次坑不如就採一次，於是我們決定使用跨平台的 engine。

  1. 在來要用什麼樣的引擎??

這問題其實超難對於新手來說，要考慮學習成本，考慮人材問題（總不能選個較冷門的然後招不到開發者吧？），然後考慮效能（這邊只能猛看文章惹）以及可能的開發時間。

這篇有一篇文章在描敘各種引擎的比較

  • 考慮學習成本包含了 文檔的完善，以及社區活不活躍，還有對於我們 team 來說最重要的是要有中文文檔 QQ。

中間歷經多次的討論，多番折騰之後我們後來選擇了 白鷺 Egret

  1. 接著語言

    白鷺 Egret 5.0.x 之後導入 Typescript以及主要的開發語言，因為有了程式底子其實上手不難，但我在這邊除了強烈推薦看完官方文檔之後（雖然我只看了一半..）另外分享我一個小捷徑，就是使用 linter 來加速學習。

    其實每個語言都有自己專屬檢查自我代碼風格的工作，關鍵字是 lint，swift 有 SwiftLint，css 有 stylelint 當然 Typescript 也有 Tslint。如果你發現那個語言沒有 linter 的話，趕快去模仿各大 open source 然後去 Github 騙星星 XD.

    為什麼我要推薦這個呢？除了到底 {} 要不要換行之外這種小風格問題，它會告訴哪些寫法不好，你第一眼一定不知道他為什麼不好，但是細看下去你會慢慢更了解這語言。

  2. 再來是框架

  3. 最後是 Egret 學習之路
