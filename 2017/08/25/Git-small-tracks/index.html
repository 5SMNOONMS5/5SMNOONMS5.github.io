Git small tracks

Aug 25, 2017

Git 小方法

git shallow clone

最近公司配了一台新電腦，當要重新 clone 一個專案的時候發現 clone 了超久，於是就找有沒辦法方法可以快一點，發現了 *git shallow clone *幾個小招數

  • 只允許 clone 特定數量的 commits
  • 單純 clone 特定的 branch

給UI設計師們的git

假如要透過 git 來管理龐大數量的檔案的時候 ex: 比如圖片或者影片…等。總不能每改一次就重新上傳一次吧，這時候我推薦使用 Git Large File Storage，他原理就是在目錄底下存了一個 pointer file，指向真正存儲這些檔案的地方，當你真正在 clone 這些檔案的時候 git 會自動 mapping 檔案的所在目的地。

安裝

指定哪些檔案你要(track)跟蹤的，.gitattributes 記得也要跟蹤

然後結束，接著就可以 push 了
